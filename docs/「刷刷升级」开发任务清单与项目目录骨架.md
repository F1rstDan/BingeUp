# 「刷刷升级」(BingeUp) Beta v0.1 开发任务清单与项目目录骨架

## 一、开发目标

第一版交付一个可供少量真实用户测试的 Chromium 浏览器插件，支持：

* Chrome、Edge；
* Bilibili 普通视频和竖屏视频；
* YouTube 普通视频和 Shorts；
* 用户主动添加其他网站；
* 识别视频并覆盖题目；
* 暂停和恢复视频；
* 全局冷却；
* 新词学习、单词复习；
* 连续学习模式；
* 本地词库、本地调度、本地存储；
* 安装引导、插件面板、设置页和学习统计。

核心流程：

```text
页面加载或视频切换
        ↓
检查网站是否启用
        ↓
检查全局冷却
        ↓
识别视频区域
        ↓
暂停视频并展示题目
        ↓
提交 / 提交并继续 / 跳过
        ↓
保存学习结果
        ↓
恢复视频并更新冷却
```


规则：

### 正常提交




```text
nextAllowedAt = 当前时间 + 60 分钟
```

## M4-03 播放控制器

```ts

  playbackRate: number;
```

规则：

* 弹题前播放：答题后恢复；

* 恢复播放失败：不重复调用，不进入无限重试；

* 今日已展示过的词；
* 超出用户目标难度的词。

## M9 验收标准

* [ ] 同一个新词不会被重复创建；
* [ ] “知道了”后十分钟内不测试；
* [ ] 到十分钟后只在自然触发点测试；
* [ ] “我认识”后不会立刻再次出现；
* [ ] 每日新词上限严格生效。

## M10：题目生成器

## M10-01 英文选中文

输入：

* 当前目标词；
* 相同词性、相近难度的候选词。

输出：

```ts
interface MultipleChoiceQuestion {
  id: string;
  type: QuestionType;
  cardId: string;
  prompt: string;
  options: QuestionOption[];
  correctOptionId: string;
  explanation: QuestionExplanation;
}
```

## M10-02 中文选英文

要求：

* 中文释义不能过于宽泛；
* 其他英文选项与正确答案难度相近；
* 避免同义词同时出现。

## M10-03 例句语境题

要求：

* 例句中目标词被替换为空格；
* 词形必须与句法匹配；
* 干扰项词性一致；
* 不能通过单复数或时态明显排除。

## M10-04 拼写题

只在连续模式中使用。

第一版允许：

* 完整输入；
* 忽略首尾空格；
* 默认不忽略拼写错误；
* 大小写不敏感。

## M10-05 题型选择

建议规则：

```text
新词第一次测试 → 英文选中文
短期阶段后期 → 中文选英文
长期复习 → 三种选择题轮换
连续学习模式 → 可加入拼写题
```

## M10 验收标准

* [ ] 每道选择题只有一个正确答案；
* [ ] 四个选项不重复；
* [ ] 正确答案位置随机；
* [ ] 同一单词连续复习时不总是同一题型；
* [ ] 拼写题不会出现在普通弹题中。

## M11：答题评分与 FSRS

## M11-01 自动评分

输入：

* 是否答对；
* 答题时间；
* 是否更改过选项；
* 历史记录；
* 用户纠正操作。

第一版建议阈值：

```text
答错 → Again

答对且耗时明显较长 → Hard

正常速度答对 → Good

连续多次快速答对 → Easy
```

阈值统一放在配置文件，禁止散落在代码中。

## M11-02 短期阶段调度

新词未进入长期阶段时使用简单规则。

示例：

```text
首次展示
    ↓ 10 分钟后
首次测试

答错 → 再次设置短期到期时间
答对 → 进入长期 FSRS 阶段
```

## M11-03 FSRS 调度器

封装独立接口：

```ts
interface ReviewScheduler {
  schedule(
    card: CardRecord,
    rating: AnswerRating,
    reviewedAt: Date
  ): CardRecord;
}
```

业务代码不能直接依赖 `ts-fsrs` 的内部字段。

这样后续更换库或升级版本时，只修改适配层。

## M11-04 用户纠正评分

### “其实是蒙的”

* Good 降为 Hard；
* Easy 降为 Hard 或 Good；
* 重新计算下次复习时间。

### “这个太简单”

* Good 提升为 Easy；
* 重新计算下次复习时间。

## M11 验收标准

* [ ] 答错后的下次复习时间缩短；
* [ ] 答对后的间隔逐渐增长；
* [ ] 修改评分后调度同步更新；
* [ ] FSRS 状态可持久化；
* [ ] 升级页面后结果不丢失。

## M12：连续学习模式

## M12-01 进入方式

用户在普通题目中选择：

> 提交并再来几题

处理：

* 提交当前答案；
* 不恢复视频；
* 进入连续模式；
* 加载下一题。

## M12-02 页面结构

上方：

* 上一道题正确或错误；
* 用户答案；
* 正确答案；
* 简短解释；
* 浅绿或浅红背景。

下方：

* 下一道题；
* 选项或输入框；
* 操作按钮。

## M12-03 连续模式按钮

### 提交

小字：

> 并返回视频

行为：

* 提交当前答案；
* 退出连续模式；
* 恢复视频；
* 显示反馈小窗；
* 更新正常冷却。

### 提交并继续

行为：

* 提交当前答案；
* 更新上方反馈；
* 加载下一题；
* 视频保持暂停。

### 结束学习

行为：

* 不提交当前题；
* 退出连续模式；
* 恢复视频；
* 按正常完成还是跳过处理，需要统一产品规则。

建议第一版：

> 结束学习不算跳过，使用默认冷却。

## M12-04 连续模式出题优先级

1. 到期且近期错误的词；
2. 普通到期词；
3. 自报认识验证词；
4. 已经展示且进入短期阶段的新词；
5. 当日允许的新词。

## M12 验收标准

* [ ] 连续答题期间视频保持暂停；
* [ ] 上一道反馈与下一题同时存在；
* [ ] 提交后正确退出并恢复视频；
* [ ] 结束学习不会保存未提交答案；
* [ ] 连续模式不会无限加入新词。

## M16-02 三级兼容判断

### 完整适配

* 找到主视频；
* 可以获取区域；

### 通用视频模式

```text
冷却已结束

```

不要分析滚动速度、阅读行为或用户注意力。

## M16 验收标准

## M18：稳定性与错误处理

## M18-01 视频恢复保护

必须防止视频永久暂停：

* Overlay 卸载时尝试恢复；
* React 出错时执行清理函数；
* 提交异常时允许用户关闭遮罩；
* 恢复失败时显示“返回视频”按钮；
* 不做无限重试。

## M18-02 内容脚本错误隔离

* 每个 Adapter 独立捕获错误；
* B 站适配失败时退回 GenericVideoAdapter；
* Generic 失败时退回 BasicWebAdapter；
* 一个模块失败不能阻止 Popup 和设置页运行。

## M18-03 本地数据库异常

* 数据库打开失败时显示可理解提示；
* 提供导出和重建数据库入口；
* 不静默清空数据。

## M18-04 性能

* 禁止高频全 DOM 扫描；
* MutationObserver 回调节流；
* 页面隐藏时停止非必要定位更新；
* 题目资源按需加载；
* 词库不整体常驻内存。

## M18 验收标准

* [ ] Adapter 报错后页面仍可正常看视频；
* [ ] 遮罩异常时可以手动关闭；
* [ ] 插件不会持续高占用 CPU；
* [ ] 不存在无限 MutationObserver 循环；
* [ ] 不存在重复 React Root。

# 六、关键模块职责

## `content-controller.ts`

当前网页的总控制器，仅负责协调：

```text
Adapter 检测到视频变化
        ↓
TriggerController 判断是否应弹题
        ↓
暂停视频
        ↓
OverlayController 打开界面
        ↓
提交结果
        ↓
恢复视频
```

不要在这里实现：

* FSRS 算法；
* 数据库细节；
* 网站选择器；
* 干扰项生成；
* 冷却计算规则。

## `trigger-controller.ts`

只回答一个问题：

> 当前事件是否允许弹题？

检查：

* 网站是否启用；
* 插件是否暂停；
* 是否首次触发；
* 全局冷却是否结束；
* 是否是新视频；
* 是否是广告或预览；
* 是否符合直播规则。

## `question-service.ts`

负责：

* 获取下一张学习卡；
* 决定新词展示还是题目；
* 选择题型；
* 生成问题；
* 提交答案；
* 更新调度；
* 写入日志。

## `learning-priority.ts`

只负责学习队列排序：

```text
近期错误到期词
普通到期词
自报认识验证词
短期新词
新词
```

## `fsrs-adapter.ts`

隔离第三方 FSRS 库。

其他业务模块只使用自己的 `ReviewScheduler` 接口，不直接导入 `ts-fsrs`。

## `adapter-registry.ts`

根据当前网址选择：

1. BilibiliAdapter；
2. YouTubeAdapter；
3. GenericVideoAdapter；
4. BasicWebAdapter。

## `overlay-controller.ts`

负责：

* 创建 Shadow DOM；
* 挂载 React；
* 更新位置；
* 关闭 React Root；
* 清理事件；
* 异常时恢复视频。

# 七、建议实现顺序

不要同时开发全部页面。按以下顺序推进：

```text
1. 插件工程启动
2. Content Script 找到视频
3. 暂停和恢复视频
4. 视频区域遮罩
5. 硬编码一道测试题
6. 提交、跳过和冷却
7. B 站适配
8. YouTube 适配
9. IndexedDB
10. 内置小型测试词库
11. 新词和选择题
12. 短期调度
13. FSRS
14. 连续学习模式
15. 安装引导和 Popup
16. 设置页和统计
17. 自定义网站
18. 数据导入导出
19. 完整 Beta 测试
```

在第 6 步完成以前，不要先开发：

* 统计图表；
* 精细安装引导；
* 大型完整词库；
* 自定义网站；
* Anki 数据兼容代码；
* 复杂动画。

# 八、第一条可运行纵向切片

第一阶段不要追求全部功能，应先完成一条完整路径：

```text
打开 Bilibili
    ↓
找到主视频
    ↓
暂停视频
    ↓
显示硬编码选择题
    ↓
用户选择答案
    ↓
点击提交
    ↓
关闭遮罩
    ↓
恢复视频
    ↓
进入两分钟全局冷却
```

这一条链路稳定后，再把硬编码题目替换成真实学习系统。

纵向切片验收标准：

* [ ] 页面刷新后自动生效；
* [ ] 遮罩只覆盖视频；
* [ ] 视频能正确暂停；
* [ ] 提交后能正确恢复；
* [ ] 跳过后能正确恢复；
* [ ] 两分钟内刷新其他已启用页面不会再弹；
* [ ] 插件重新加载后不会导致视频永久暂停。

# 十、Beta 开发完成定义

只有同时满足以下条件，任务才算真正完成：

* 功能代码已经合并；
* TypeScript 无错误；
* 核心单元测试通过；
* 手动验收通过；
* 异常路径有处理；
* 数据不会因刷新或重启丢失；
* 不会导致视频永久暂停；
* 已在 Chrome 和 Edge 测试；
* 对应文档已更新；
* 没有把 P2 功能偷偷塞进第一版。
